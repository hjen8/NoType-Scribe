# Typeless Scribe (NoType)

## 專案知識沉澱 (Knowledge Sedimentation)

### ⚠️ 開機自動啟動機制的重要提醒
本專案的「開機自動隱藏啟動」高度依賴 Windows 工作排程器 (Task Scheduler) 與 VBS 腳本的配合。
- **核心路徑**：工作排程器任務 `\NoType_AutoStart` 綁定執行 `run_hidden.vbs`。
- **啟動鏈條**：`Task Scheduler` -> `run_hidden.vbs` -> `start_admin.bat` -> `pythonw.exe main.py`
- **防呆警告**：在執行「收工清理 (Cleanup)」時，**絕對不可將 `run_hidden.vbs` 或 `start_admin.bat` 視為無用暫存檔刪除**。若將其移除，將會導致隔日開機時工作排程器找不到檔案而跳出 Script 錯誤。

### 🧠 專案初創架構與核心設計決策 (Genesis Architecture)

這部分記錄了本專案在初始開發時，針對使用者獨特的工作環境與硬體配置所做的關鍵決策：

#### 1. API 引擎選擇：為何選用 Groq 而非本機 GPU 或 Gemini？
- **極致的低延遲需求**：語音打字輔助工具要求「毫秒級」的回饋。純地端運算（即便有 RTX 4080）處理 STT+LLM 仍可能耗時近 10 秒，無法滿足即時打字需求。
- **LPU 架構優勢**：Groq 採用專為大語言模型推理設計的 LPU (Language Processing Unit)，初次延遲 (Time to First Token) 極低，能有效消除打字時的「頓挫感」。
- **適才適所策略**：將「高頻、輕量」的打字語音辨識與錯字修飾交給 Groq (`whisper-large-v3` + `llama3-70b-8192`)；而將強大的 Gemini API 額度保留給需要深度思考的備課與分析任務。

#### 2. UI 介面與多螢幕防護 (4 螢幕 VESA 陣列)
- **座標鎖定防呆**：由於使用者的桌面是由四台螢幕組成的陣列，所有的浮動 UI（如「錄音中...」提示、系統匣圖示、設定視窗）**必須強制鎖定顯示於主螢幕 (Primary Display) 的絕對座標**。嚴禁視窗生成於延伸螢幕邊緣，以免視窗迷失。

#### 3. 快捷鍵與輸入法防禦機制
- **避讓機制**：觸發錄音的快捷鍵 (預設 `Alt + Space`) 必須實作嚴謹的鍵盤 Hook，確保絕對不會與使用者常駐的 **WeType 輸入法** 或是 **XMind 8** 產生熱鍵衝突。
- **Timeout 解鎖**：必須具備 Timeout 自動解除鎖定的防呆機制，避免程式異常時造成鍵盤全域鎖死。

#### 4. 專屬語意修飾 Prompt (高中地理教師情境)
- 為了達到最高的修飾精準度，送給 LLM 的 System Prompt 必須針對使用者身分特化：
  > 「使用者是一位高中地理教師，內容常涉及『探究與實作』課程、SDGs、GIS 系統操作、以及高中地理專有名詞。請自動過濾掉口語贅詞，並直接輸出修正後的純文字，絕對不要包含任何解釋或開場白。」

#### 5. 字典檔替換防過度糾正 (Over-correction Guardrails)
- **問題現象**：當向 LLM 下達「極度嚴格強制替換同音字」的指令時，如果使用者說出字典中沒有的詞彙（如「林志強」），LLM 會為了強行套用規則，將其強制篡改為發音略為相似的字典詞彙（如「林政弘」）。
- **防呆心法**：在 System Prompt 中，除了提供字典外，必須同時加上明確的**安全護欄警告**：「如果語音辨識的發音跟字典明顯不同，請保持原樣，絕對不可強行套用字典！」以避免 AI 過度積極干預導致語意扭曲。

#### 6. 單句無句號優化 (Single Sentence Punctuation Optimization)
- **設計決策**：為了方便使用者在輸入短片語或人名後接續打字，已在 System Prompt 中加入「單句無句號」規則。只要輸入的是「單一一個句子」（中間無逗號斷句），AI 就絕對不會在句尾加上句號；若為兩句以上的複合句，才會加上句號，完全符合人類直覺。

#### 7. 全域熱鍵與 UIPI 權限防禦 (UI Privilege Isolation)
- **問題現象**：當程式以一般權限執行時，若使用者在「系統管理員權限」的視窗 (如 CMD 或 Antigravity) 內按下快捷鍵 (F9)，會因為 Windows UIPI 安全機制而導致全域熱鍵失效。
- **防呆解法**：`start_admin.bat` 腳本已加入**自動提權 (UAC)** 機制 (`net session` 判斷與 `getadmin.vbs` 呼叫)。確保使用者手動雙擊啟動時，會自動彈出權限確認視窗，獲取最高權限後再啟動 Python 程式，藉此穿透 UIPI 限制。

#### 8. 🚫 LLM 過度貼心防呆：專有名詞的擴充與翻譯
- **現象與問題**：遇到英文專有名詞 (如 `Skill`) 或縮寫 (如 `SOP`) 時，LLM 常會出於「假貼心」自動翻譯或擴充全名。即使一般性地要求「保留英文」，LLM 仍可能自作主張加上括號解釋。
- **防呆心法**：在 System Prompt 中，必須採用**「極度嚴厲且具體點名」**的方式禁止（例如：絕對不可寫成「技能(Skill)」或「標準作業程序(SOP)」）。
